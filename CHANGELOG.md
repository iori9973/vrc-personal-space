# Changelog

## [0.12.0]

- **同一プロジェクトで複数アバターに適用できないバグを修正**。生成アセット（コントローラ/メニュー/クリップ/マテリアル）が全アバター共通の固定パスだったため、2体目をセットアップすると1体目のアセットを削除・上書きして壊していた（最後の1体しか効かない）。**アバター名ごとのサブフォルダ**（`Assets/PersonalSpace/Generated/<アバター名>/…`）に分けて解決。
  - リングメッシュ（形状共通）のみ `Generated/PS_RingMesh.asset` として全体共有。
  - 既存プロジェクトは各アバターで「セットアップ / 更新」を1回押し直すと新レイアウトに移行する（旧 `Generated/` 直下の共通アセットは未使用の残骸になるので手動削除可）。
  - 注意: アバター名が完全に同じ2体は同じフォルダを共有してしまうため、名前を分けること。

## [0.11.0]

- **透明化の距離も可視化リングを表示**。透明化ページに「範囲表示」トグル（`PS_ShowCloak`）を追加し、`PS_CloakRange` に連動する足元リングを表示（自分のみ）。
  - **色分け**: 押し出し＝**イエロー**、透明化＝**シアン**。両方使うと足元にリングが2つ出る。
  - 範囲表示リングの生成処理を共通化（押し出し用・透明化用）。

## [0.10.0]

- **透明化の距離をメニューから調整可能に**。Expression Menu に「透明化の距離」Radial（`PS_CloakRange`・ローカル/非同期）を追加。近接センサーの半径をアニメで動的に変え、`透明化する最大距離`の範囲内で実行中に絞れる（押し出しの「反応範囲」と同じ考え方）。
- **メニューを機能ごとのサブページに整理**。押し出し・透明化の両方が有効なときは「押し出し」「透明化」の2ページに分割。片方だけなら従来どおり直接並べる。
  - Setup の「透明化する距離」表記を「透明化する最大距離」に変更（メニューでこの範囲内に絞る）。
- **「自分からも消える」トグルを追加**（透明化ページ・`PS_CloakSelf`・ローカル/非同期）。通常は他人視点だけ消える（自分には見える）が、ONにすると自分視点でも消える。既定 OFF。

## [0.9.0]

- **Hierarchy の生成物を1つに集約**。これまでアバター直下に4つ（`PersonalSpaceSensors` / `PS_RangeViz` / `PS_NearSensor` / `PS_ModularAvatar`）分かれていたのを、単一の **`PersonalSpace`** コンテナ配下（`Sensors` / `PS_RangeViz` / `PS_NearSensor` / `PS_ModularAvatar`）にまとめた。Hierarchy がすっきりする。
  - 旧レイアウト（直下に個別配置）はセットアップ/削除時に自動で掃除する。
  - 範囲リングのアニメ参照パスをコンテナ配下（`PersonalSpace/PS_RangeViz`）に合わせて更新。
  - 反映には**もう一度「セットアップ / 更新」→ 再アップロード**が必要。

## [0.8.1]

- **Expression Menu の中身（Personal Space サブメニュー）が空になり、実機で項目が出ないバグを修正**。セットアップ時に「メニューアセットを削除 → 同フレームで作り直す」順序になっており、Unity の AssetDatabase 競合で `PS_Menu` の controls が空のまま保存されることがあった。メニューは削除せず中身をその場で作り直す方式にして解消（「削除」ボタン時のみ完全削除）。
  - 影響を受けた方は、パッケージ更新後にもう一度「セットアップ / 更新」を押してから再アップロードしてください。

## [0.8.0]

- **OSC アプリに GUI を追加**。`osc/PersonalSpace.bat`（または `python personal_space_gui.py`）で、起動状態のランプ（停止中／接続中／待機中／動作中）・ログ・現在の逃走ベクトル/移動量をウィンドウで確認できるようになった。開始/停止ボタン、センサー数・強さ・平滑化などの設定もウィンドウから操作可能。
  - CLI（`python personal_space.py`）は従来どおり利用可能。共通ロジックを `PersonalSpaceWorker` に切り出して両対応にした。
- **VRChat 連動の自動起動**を追加。GUI の「Windows起動時に自動で常駐」を ON にすると、常駐ウォッチャ（`personal_space_watch.py`）がスタートアップに登録され、**VRChat を起動するとアプリが未起動なら自動で立ち上がり OSC を開始**する。GUI は二重起動しない（単一インスタンス化）。
- **OSCアプリを PC 共通の固定フォルダにインストールするボタン**を Setup Window に追加。`%LOCALAPPDATA%\VRCPersonalSpace\app\` へ osc 一式をコピーし、複数プロジェクトでもアプリの実体を1箇所に集約。自動起動のパスもプロジェクトに依存しなくなる（`Packages/` 内から直接自動起動を登録しようとすると警告）。
- **透明化する距離の上限（1.5m）を撤廃**。数値入力欄にし、押し出しと同様に上限なしで指定できるようにした。

## [0.7.0]

- **機能を単体で選べるように**。押し出し機能と透明化機能をそれぞれ独立したチェックボックスにし、**透明化だけを ON**（OSC アプリ不要のアバター単体運用）や**押し出しだけ**のセットアップが可能になった。
  - Setup Window の「機能」欄で `押し出し機能（要OSCアプリ）` / `透明化機能（近づかれたら消える）` を個別に切り替え。オフにした機能のセンサー・パラメータ・メニュー項目は生成されない。
  - セットアップは毎回いったん全生成物を撤去してから、有効な機能だけを作り直す方式に統一（設定変更が確実に反映される）。
  - メニューはオンにした機能の項目だけを追加（透明化のみなら「透明化」トグルだけ）。
- **範囲表示を塗りつぶし円 → リング（輪）に変更**。足元の反応範囲を、中が抜けた輪郭のリングで表示するようにした（自分視点の視界を塞ぎにくい）。

## [0.6.1]

- 範囲表示/透明化のセットアップ時に `MissingComponentException`（PS_RangeViz の MeshFilter 等）で失敗するバグを修正。`GetComponent() ?? AddComponent()` が Unity の疑似 null を無視する問題を `if (== null)` に置換。

## [0.6.0]

- **透明化モード**を追加。近づかれると**他人視点で自分のメッシュが消える**（自分視点では見えたまま）。メニュー「透明化」ON で有効。押し出しの代替として使える。
  - 仕組み: 近接検知の Contact Receiver → 同期パラメータ `PS_Near` → リモート(IsLocal:false)で全 Renderer を非表示。OSC アプリ不要のアバター単体ギミック。
  - 制約: VRChat の仕様上、特定の人だけに消えることはできず**全員から消える**。

## [0.5.0]

- **センサーを縦カプセル化**。高さ方向を広く覆うため、身長差のあるアバターでもすり抜けにくくなった（水平距離ベースで検知）。
- **反応範囲の視覚表示**を追加。足元に半透明ディスクを表示し、メニュー「範囲表示」ON で `PS_Range` に連動して大きさが変わる（自分のみに表示・ローカル）。

## [0.4.1]

- 「削除」ボタンで `PS_ModularAvatar`(MA コンポーネント一式)と生成メニュー資産(PS_Menu/PS_MenuRoot)が消えなかったバグを修正。空の MA Parameters が残って統合オブジェクトが削除されない問題も解消し、完全にクリーンアップするように。

## [0.4.0]

- **1ウィンドウ・1ボタンに統合**。Setup Window の「セットアップ / 更新」だけで、センサー＋遅延補償＋メニューを一括生成するように変更（旧「Remote Offset」ウィンドウは廃止）。
- 上級者向けに「遅延補償を含める」「メニューを追加する」のチェックボックスで個別オフも可能。

## [0.3.1]

- 旧バージョン(〜0.2.0)がアバターの Expression Menu に直接差した「Personal Space」submenu を、セットアップ時に自動で掃除するように修正（`CleanupLegacyMenu`）。これがないと未定義パラメータ警告やメニュー重複が残っていた。

## [0.3.0]

- **Modular Avatar 対応（非破壊注入）**。パラメータ・メニュー・遅延補償アニメを、Expression 資産を直接編集する代わりに MA コンポーネント（MA Parameters / MA Menu Installer / MA Merge Animator）でアバター直下の `PS_ModularAvatar` に注入するように変更。
  - **FaceEmo など NDMF 系ツールと共存**でき、ビルド時に項目が消える問題を解消。
  - リモート位置ズラしの Controller が**自動で FX にマージ**される（手動 Merge Animator 設定が不要に）。
  - メニューは「Personal Space」サブメニューとして追加。
  - Modular Avatar (>=1.10.0) が必須依存に。

## [0.2.0]

- 方向センサーの既定を 16 に変更（最大反応半径の既定も 1.5m に）。
- Expression Menu に実行時調整の Radial を追加：反応範囲 `PS_Range` / 押し出しの強さ `PS_Gain` / 遅延補償の量 `PS_Lead`（いずれも非同期、アプリが購読して反映）。
- 反応範囲は物理半径を最大にしたまま近接度しきい値で縮める方式（アニメ生成不要）。

## [0.1.0]

- 初回リリース。
- N 方向 Contact Receiver による他プレーヤーの接近検知（Local Only, ランク非計上）。
- OSC アプリによる自動回避（逃走ベクトル合成 → `/input` 移動）。
- OSCQuery による自動接続（固定ポートへフォールバック可）。
- リモート位置ズラし（遅延補償）。
- Expression Menu による ON/OFF。
