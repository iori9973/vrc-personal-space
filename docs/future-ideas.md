# 将来の機能アイデア

## 通行証アイテム方式のホワイトリスト（特定人物だけ近づける）

> **v0.15.0 で実装済み。** 使い方は README「通行証機能」を参照。以下は設計メモ。
> 実装は「方向別キャンセル（通行証所持者がいる方向だけ押し出しを打ち消す）」＋「合言葉設定式」＋「Prefab 書き出し」。

### 背景
Contacts も OSC も「誰が近づいたか」の身元情報を持たないため、フレンド判定や名前指定は不可能。
ただし **Contacts のカスタム衝突タグは異なるアバター間でも一致すれば反応する**ため、
「共通の秘密タグを持つ人だけ検知」＝“所持ベースのホワイトリスト”は実現できる。

### 仕組み
- **通行証アイテム（配布側）**: ピアス/指輪/バッジ等どんなアクセサリでもよい（不可視でも可）。
  そこに **VRCContactSender**（共通の秘密タグ、例 `PSPass_<合言葉>`）を仕込む。
  MA Prefab として配布し、相手はアバター直下にドラッグするだけ。
- **本ツール側**: その秘密タグ専用の **VRCContactReceiver** を追加 → パラメータ `PS_Pass` を立てる。
  アプリが `PS_Pass` を読み、**パス所持者が近い間は押し出しを止める**。
- **メニュー**: モード切替「全員から離れる / 通行証は通す / OFF」。

### 実装に必要なもの
1. ツール: 通行証タグ設定（任意の合言葉文字列）＋受信機＋`PS_Pass` パラメータ＋メニューのモード。
2. アプリ: `PS_Pass` 購読 → 「通行証は通す」モード時、`PS_Pass` が高ければ出力 0（押し出さない）。
3. 配布用 **通行証アイテム Prefab**（Contact Sender 入り、MA で着けるだけ）。

### 注意点・トレードオフ
- **身元ではなく“所持”ベース**。アイテムを渡した人は全員通れる（仲良しグループ向けには最適）。
- タグ文字列が漏れると他人も通れる → タグは信頼できる人にだけ配る前提（アップロード済みアバターの
  タグは他人から簡単には読めないので実用上そこそこ安全）。
- 通行証所持者と他人が同時に近いと、単純実装では他人も押し出さなくなる（フレンドを迎える用途なら許容）。
  方向ごとに厳密分離するのは複雑。
- 相手の協力が必要（相手も通行証アイテムを入れる）。

### 補足（不可能なこと）
- 任意の相手をフレンド/名前で自動判定するのは原理的に不可能（Contacts/OSC に身元情報なし）。
- OSC は自分のアバター情報しか出さない。ログファイルは在席名簿は取れるが空間位置は取れない。
- 「フレンドは通す/この人は通さない」は VRChat 標準のセーフティ(Avatar Interactions 個別/ランク別)で対応可。
